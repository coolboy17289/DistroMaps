# DistroMap Refactoring Plan — Production-Grade Linux Ecosystem Knowledge Platform

## Current State Analysis

### What Works (Keep)
- **TypeScript autonomous crawler** (2,259 lines) — 8 sources, trust scoring, query expansion, ISO probing
- **505 distros across 88 families** in `data/distros.json`
- **Vue 3 + Three.js 3D frontend** (`frontend-vue/`) — working 3D brain graph with OrbitControls
- **React 2D Canvas frontend** (`frontend/`) — production SPA on Vercel
- **Vercel serverless API** (12 endpoints) — search, path, compare, stats, families
- **Express backend API** (`backend-api/`) — PostgreSQL-backed routes, Redis caching
- **PostgreSQL schema** (`backend-api/db/init.sql`) — entities, edges, embeddings, releases
- **Docker Compose** — 5 services (postgres, redis, api, frontend, scrapers)
- **Python Scrapy skeleton** — items, pipelines, settings (no spiders yet)
- **Discovery engine** (`scripts/discover-distros.ts`) — Wikipedia pattern learning

### What's Broken/Missing
1. **Scrapy spiders** — `backend-data/scrapers/spiders/` is empty; Dockerfile runs non-existent spider
2. **AI processor** — referenced in docker-compose but `workers.ai_processor` doesn't exist
3. **Embeddings** — pgvector table exists, no code populates it
4. **Self-learning keyword expansion** — crawler has query expansion but not dynamic learning
5. **Logo/image management** — `logo_url` column exists, no scraper fetches logos
6. **Release history** — `releases` table exists, no population code
7. **No CI/CD** — no `.github/` directory
8. **No tests** — no test files anywhere
9. **No documentation** — no README, ARCHITECTURE, etc.
10. **Unused Vue deps** — graphology, sigma, gsap installed but not imported

### Architecture Decision: Two Frontends → One Consolidated App

**Decision:** Build a new **Next.js 14+ app** (`app/`) as the primary frontend, consolidating:
- The Vue 3D graph rendering → port to **React Three Fiber** + **@react-three/drei**
- The React search/filter/side-panel UI → port to Next.js App Router
- The Vercel serverless API → Next.js API routes
- The Express backend API → merge into Next.js route handlers

**Keep `frontend-vue/` and `frontend/` as legacy reference** — do not delete.

### Phase 5: AI Processing (Files: 4, ~800 lines)

**Goal:** Implement AI-powered data processing pipelines.

**Create in `backend-data/workers/`:**

1. `ai_processor.py` — Main AI worker:
   - Loads new distros from database
   - Generates embeddings (sentence-transformers or OpenAI)
   - Stores in pgvector embeddings table
   - Runs duplicate detection
   - Discovers relationships

2. `embeddings.py` — Embedding generation:
   - Sentence-transformers for local processing
   - OpenAI ada-002 as fallback
   - Batch processing for efficiency
   - Cache embeddings to avoid re-computation

3. `relationship_discovery.py` — Auto-discover relationships:
   - Parse descriptions for "based on", "derivative of"
   - Package manager family detection
   - Similar distro recommendations via vector similarity

4. `duplicate_detection.py` — Find duplicates:
   - Levenshtein distance on names
   - URL normalization
   - Description similarity via embeddings

## Implementation Order

Execute phases sequentially. Each phase builds on the previous.

**Estimated total:** ~7,700 lines of new code across 60+ files.

**Critical path:** Phase 1 → Phase 2 → Phase 3 → Phase 6 (minimum viable product)

**Parallelizable:** Phase 4 (scrapers) can run alongside Phase 2-3. Phase 5 (AI) needs Phase 4 output. Phase 7-8 (deployment/docs) can run alongside anything.

---

## Risk Mitigation

1. **Keep legacy frontends** — Don't delete `frontend/` or `frontend-vue/` until new app is verified
2. **Incremental migration** — Each phase is independently deployable
3. **Data safety** — `data/distros.json` remains the source of truth; PostgreSQL is derived
4. **Performance** — Use InstancedMesh for 500+ nodes; lazy-load graph data
5. **Rate limiting** — Respect Wikipedia/GitHub APIs; use existing trust scoring system
